Objetivo: Hacer un modal panel de tipo "Si o No" super sencillo [menos de 5 minutos] y finalmente no repetir código.
Descripción: cuando se tiene más de un modal con el mismo controlador, es necesario que se identifique a que modal
se le esta dando "Yes", esto se hace con el parámetro que se recibe (idModal)
Precondiciones:
1. Asegurarse que aparezca: xmlns:c="http://java.sun.com/jstl/core"

Uso:
1. Copiar uno de los ejemplos de abajo y cambiar los parámetros.
2. La controladora que tiene el método que responde al click del botón "Yes" tiene que heredar de la clase
GenericYesNoModalPanel (gehos.ensayo.codebase.genericComponents.yesNoModal.GenericYesNoModalPanel) y sobreescribir el 
método[public abstract void ExecuteYes(String modalId)]
3. En el <a4j:commandLink> el onclick="#{nombreDelModal}('#{id_del_objeto}')"

Ejemplo de implementación del método:
public void ExecuteYes(String modalId)
{
	if(modalId.endsWith("modal_eliminar_regla"))
	{
		Object o = super.GetObjectIdForModalId(modalId).getValue();
		if(o!=null)
		{
			Long ruleId = Long.parseLong(o.toString());
			Regla_ensayo r = entityManager.find(Regla_ensayo.class, ruleId);
			entityManager.remove(r);
			entityManager.flush();
			RefreshRuleList();
		}
	}
	else if(modalId.endsWith("modal_completar_regla"))
	{
		CompleteRules();
	}
			
}
